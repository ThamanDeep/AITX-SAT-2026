How the system learns

Seven quiet hours.
One safer policy by morning.

Every night, commerce agents test new ways to retrieve evidence and make buy-or-wait decisions. Nemotron teaches from failures; Verifiers decides what actually improved.

The nightly evolution loop

12:00 AM → 7:00 AM

Production target · current repository behavior remains a simulation until this pipeline is fully wired.

  1. 12:00

    Freeze the world

    Snapshot listings, policy version, golden dataset, and retrieval index so every candidate receives the same evidence.

  2. 1:00

    Agents explore

    Scout, Inspector, Speculator, and marketplace agents browse allowed sources, retrieve URLs, and run purchase cases.

  3. 3:00

    Verifiers score

    Rubrics grade decision accuracy, landed price, source validity, retrieval recall, latency, hallucination, and deadline regret.

  4. 4:30

    Nemotron teaches

    The main teacher reviews failure traces and proposes one focused change to the prompt, tool routing, memory, or retrieval policy.

  5. 5:30

    Challenge the candidate

    Held-out cases, hard-case mining, memory-on/off ablations, and rolling-origin price backtests check whether the gain is real.

  6. 6:30

    Promote or roll back

    The challenger becomes champion only if confidence, safety, URL validity, and regression gates all pass.

  7. 7:00

    Publish the morning policy

    Version the winning policy, attach an evaluation report, update the leaderboard, and keep the prior champion ready for rollback.

The evaluation contract

Reward evidence, not eloquence.

Prime Intellect Verifiers packages each task with a dataset, environment, tools, and rubric. Multiple rewards become one promotion decision.

30%

Decision quality

Correct buy, wait, or abstain call against the golden answer.

20%

Retrieval accuracy

Relevant products, correct variants, valid pages, and enough coverage.

15%

Landed-price error

Price, shipping, condition, and seller costs match checkout reality.

15%

Groundedness

Claims trace to retrieved evidence; unsupported claims fail.

10%

Latency

Useful answer within the response budget.

10%

Decision regret

Penalize waiting too long or buying before a better window.

What is real today?

The repository contains a 15-case golden dataset, scoring scripts, a coordinator, and simulated improvement iterations. It does not yet prove autonomous overnight learning. This methodology is the implementation contract: connect live Verifiers rollouts, freeze held-out tests, then publish only measured runs.
